Web设计基础

Web站点建立的流程

建立Internet站点

公司企业都需要在Internet上建立自己的企业站点，宣传企业文化、产品和在线销售等为主。这些站点的内容都为公开性质的。
Internet站点建设的主要流程如下：

  1. 了解客户需求
  2. 制定网站建设方案
  3. 双方签署建设合同
  4. 完成详细设计文档
  5. 进行网站页面设计与功能开发
  6. 网站试运行，客户验收
  7. 网站的使用培训
  8. 网站的后期维护

建立Intranet系统

即采用Internet技术的企业内部网，通过对防火墙的设置，把企业内部系统限制在企业内部访问，例如OA系统（办公自动化系统）、ERP系统等。

Web站点设计的原则

  1. 以用户为中心
  2. 易用性

Web服务器与应用服务器

Web是Internet服务中的一种，Internet的服务包括：

  • FTP
  • E-mail
  • 流媒体
  • Web

Web中定义了两个方面：客户端和服务器，其关系如下图

Web就是基于请求/响应模型的，其过程是：客户端给Web服务器发送一个请求页面，服务器接受请求，并处理请求返回一个响应给客户端，通常是一个HTML界面 。Web服务器通过响应HTML来处理HTTP请求，例如响应静态页面或图片来进行页面跳转，或者把动态响应的结果委托给服务器端程序执行。

服务器请求是不可以重用的，客户端连续发送两次相同的请求，服务器就要进行两次处理。主流Web服务器包括Apache和IIS。

应用服务器是集成了Web服务器的用于创杰、部署、运行、集成和维护多层分布企业级应用的平台。主流的应用服务器有Tomecat等。

apache到底是个什么?是软件还是全世界共用的服务器?

它是Apache软件基金会管理下的一个开放源代码的服务器软件，可以理解为电脑上的一个应用程序。简单地说它的作用就是将你的电脑变成一台服务器，让你的电脑开放特定的网络端口，用以接收来自网络上发送到这台机器的HTTP请求，对请求的内容进行处理并作出相应的响应。

HTML和CSS

HTML的重要特点就是能够把文本、图像、动画、音乐等多媒体相结合。本章将介绍如果通过Dreamweaver来制作网页

静态网页HTML简介

HTML文件名后缀是.htm或.html，纯HTML格式的网页通常被称为“静态网页”。
URL，是统一资源定位符，是网络上某一个站点或网页的完整路径

HTML网页可以插入各种动态效果，这种动态效果是视觉上的，与动态网页不是同一概念

静态网页的特点：

  1. 以呈现多媒体信息为主，本身并不能实现用户与服务器间的交互功能
  2. 静态网页文件都保存在网站服务器的硬盘空间里，每个网页都是一个独立的文件
  3. 内容相对稳定，容易被搜索引擎检索
  4. 不能直接获取数据库中的信息，当网站信息量大时制作比较困难

HTML常用标记

基本标记

一个标准的HTML页面的结构应有以下标记组成：

  1. 网页首尾
    一个完整的HTML文件首尾需要用<html>...</html>,浏览器从<html>处开始解析HTML代码
  2. 网页头部
    HTML头部对应的标记是<head>...</head>,中间可以嵌入其他标记或一些客户端动态脚本程序，如JavaScript等
  3. 网页标题
    对应标记为<title>...</title>,且置于网页头部
  4. 网页主体
    位于<body>...</body>之间的不分是网页主体

div标签和span标签

大多数 HTML 元素被定义为块级元素或内联元素。块级元素在浏览器显示时，通常会以新行来开始（和结束）

  2. 字符串型

Servelet技术

Servlet是java的一个类，主要功能是接受前段用户的请求，调用后端的逻辑处理程序，最终给客户返回响应

工作原理

Servlet容器

Servlet容器一般指Java应用服务器，当客户请求到来时，容器获取请求，然后调用某个Servlet，并把Servlet的执行结果返回给用户。
常见的容器包括Tomcat等。

Servlet生命周期

  1. Servlet容器创建一个Servlet实例
  2. 容器调用该实例的init方法初始化
  3. 当客户端向该Servlet发送请求时，容器调用此实例的service方法
  4. 在service方法中根据用户请求调用不同方法（doGet doPose
  5. 终止运行时，容器调用Servlet的destroy方法释放Servlet所占用的资源

Servlet API

Servlet容器可以创建实现ServletRequest和ServeltResponse两个接口的对象

  1. javax.servlet.http.Httpservlet
  • init
  • doGet doPost
  • service
  • destroy
  1. javax.servlet.http.HttpservletRequest
  • getParameter
  • getParameterValues
  1. javax.servlet.http.HttpServletResponse
  • sentContntType
  • getWriter
  • getOutputStream

JSP技术

Java Server Pages是一种动态网页技术，有两个主要功能：

  1. 将用户表单的输入数据和请求传递给控制层和模型层组件；
  2. 接受模型层处理完的数据显示给用户

JSP简介

工作原理

与Servlet类似

JSP的特点

HTML和JS代码的执行是直接被浏览器解析执行，源代码在客户端可以被查看；而JSP页面在服务器端被执行，因此客户端得到的结果并不包含原始的服务器端动态代码，更加安全
